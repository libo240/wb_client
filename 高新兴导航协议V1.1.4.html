1. 目前使用注意事项

二维栅格地图中只做对精度要求不高的操作，如地图查看，录制点查看，路径查看，导航到点，跑路径等功能。

目前有很多协议回复是二进制数据，使用Protobuf做外层封装。

Messae xx {
 string key = 1; //命令码
 string uuid = 2; //校验串码
 bytes  binaryData = 3; //二进制数据
 string  textData =4; //文本数据
}

websocket客户端请求时，随机生成uuid校验串码，服务端返回数据时返回相同的uuid，以便客户端校验信息，确认该返回数据是归属自己的；

文本回复信息使用以下数据结构:

xxxxxxxxxx
textData:   
{   
 "data": "",  //各个命令码自定义数据
 "successed": true,  //表示执行成功或失败 
 "error_code":  ,//
 "error_message": ""
}

其中error_code复用3.12中的状态码，error_message由导航定义，需保证和error_code意义一致

xxxxxxxxxx
WebSocket URL如下:
ip:8086/gxx-robot/nav_control 一问一答相关协议
ip:8086/gxx-robot/nav_message 导航主动上送协议

2. 建图相关API

2.1 开始扫描地图

地图扫描开始时需要确保前方5M无动态障碍物，然后匀速前行5-10m后再进行正常扫图。type：0 -新建扫描 1-扩展扫描

开始扫描地图后，该任务为最高优先级，需要手动取消或者结束才会停止该任务，其他初始化或者巡逻任务在建图过程中无法执行。

​x
request: 
key: "start_scan_map",  
textData:  
{  
    "map_name": "test",
    "type": 0
} 
​
response:
key: "start_scan_map",   
textData:
{
 "data": "",
 "successed": true
}

4.11设置导航路径组

其中points中第一个点和最后一个点代表起点和终点，中间的点代表途径点(不停, 只用于约束路线)

路径中点的类型包括选取点和录制点，因为获取录制点的坐标为全局坐标，而选取点的坐标为栅格坐标，所以这里点坐标的字段做了区分，grid_x和grid_y代表栅格坐标，name代表录制点的名字。选取点和录制点可以自由组合
